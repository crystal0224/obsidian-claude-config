# /rfp-handoff [SESSION_DIR]

현재 RFP 세션의 audit trail을 통합해서 다음 세션 진입 handoff 문서 자동 생성 + MEMORY 자동 update.

## 동작

1. SESSION_DIR 지정 없으면 `/tmp/orchestrate_runs/` 최신 `*_rfp_*` dir 자동 감지
2. 다음 source files 통합:
   - `plan.md` — Phase 0 baseline (단지명·시행사·위치·종류·문서·청중)
   - `audit_summary.md` — 누적 audit 발견
   - `decisions.md` — Crystal 결정 batch 결과 (있으면)
   - `ro_round1_report.md` — RO 통합 결과
   - `audit_d.md` — Final Auditor (있으면)
3. 산출물 생성:
   - `~/Desktop/brother/<YYYY-MM-DD>-handoff-<버전>.md` — 다음 세션 진입 prompt 형식
   - `~/Desktop/brother-sample/v*-audit-trail/handoff.md` — archive에도 복사
4. MEMORY 자동 update:
   - `~/.claude/projects/-Users-crystal-Desktop-brother/memory/reference_brother_pipeline.md` 갱신
   - 다음 세션 시작 prompt 1줄 update

## 산출물 구조 (handoff-<TS>.md)

```markdown
# Handoff — RFP <단지명> v<N> (<TS>)

## 0. 이전 세션 완료 항목
- Phase 진행 status (1A·1B·2·2.5·Final)
- 산출물 경로 (HTML / PDF / archive)
- audit critical 발견 N건

## 1. 다음 세션 첫 단계
[자동 추출된 진입 prompt]

## 2. 검증 시나리오 (이전 세션 미완료 항목)
- (audit_summary 미해결 항목 자동 추출)

## 3. 환경 점검
[표준 체크리스트]

## 4. 알려진 함정
[crystal-rfp-patterns 8개 referencing]

## 5. 새 작업 후보 (검증 통과 시)
[ro_round 산출물 의 R2 candidate 자동 추출]
```

## 명령

```bash
SESSION_DIR="${1:-$(ls -td /tmp/orchestrate_runs/*_rfp_* 2>/dev/null | head -1)}"
if [ -z "$SESSION_DIR" ] || [ ! -d "$SESSION_DIR" ]; then
  echo "❌ SESSION_DIR 없음. /tmp/orchestrate_runs/ 에 *_rfp_* run 없음"
  exit 1
fi

TS=$(basename "$SESSION_DIR")
DATE=$(date +%Y-%m-%d)
VERSION=$(echo "$TS" | grep -oE 'v[0-9]+' || echo "v?")
HANDOFF_OUT="/Users/crystal/Desktop/brother/${DATE}-handoff-${VERSION}.md"

# Phase 0 baseline 추출
SITE=$(grep -E '단지명:' "$SESSION_DIR/plan.md" 2>/dev/null | head -1 || echo "(unknown)")
LOCATION=$(grep -E '위치:' "$SESSION_DIR/plan.md" 2>/dev/null | head -1 || echo "(unknown)")
DOC_TYPE=$(grep -E '문서종류:' "$SESSION_DIR/plan.md" 2>/dev/null | head -1 || echo "(unknown)")

# Phase 진행 status 자동 계산
PHASES_DONE=$(ls "$SESSION_DIR" | grep -cE '^(audit_|worker_|ro_round)' 2>/dev/null)

# Final 산출물
LATEST_HTML=$(ls -t /Users/crystal/Desktop/brother-sample/sample-rfp-*-fulldeck.html 2>/dev/null | head -1)
LATEST_PDF=$(ls -t /Users/crystal/Desktop/brother-sample/sample-rfp-*-fulldeck.pdf 2>/dev/null | head -1)

# Handoff 작성
cat > "$HANDOFF_OUT" <<EOF
# Handoff — RFP $VERSION ($TS)

**작성**: $(date "+%Y-%m-%d %H:%M:%S KST")
**SESSION_DIR**: $SESSION_DIR
**Auto-generated by**: /rfp-handoff

---

## 0. 이전 세션 완료 항목

- Phase 진행: $PHASES_DONE files in run dir
- 단지: $SITE / $LOCATION
- 문서: $DOC_TYPE
- HTML: $LATEST_HTML
- PDF:  $LATEST_PDF

## 1. 다음 세션 첫 단계

\`\`\`bash
cd /Users/crystal/Desktop/brother
claude  # 새 process — skill registry refresh
\`\`\`

진입 후:
\`\`\`
/rfp-status  # 이전 세션 trail 확인
\`\`\`

## 2. audit_summary 미해결 항목

\`\`\`
$(cat "$SESSION_DIR/audit_summary.md" 2>/dev/null | tail -30 || echo "(no audit_summary.md)")
\`\`\`

## 3. RO Round 1B 미해결 R2 candidates

\`\`\`
$(grep -A 5 'Round 2 candidate\|R2-' "$SESSION_DIR/ro_round1_report.md" 2>/dev/null | head -20 || echo "(no ro_round1_report.md)")
\`\`\`

## 4. crystal-rfp-patterns 함정 8개 reference

\`~/.claude/skills/crystal-rfp-patterns/SKILL.md\` 참조 — 매 세션 자동 로드.

---

**End of auto-generated Handoff. Edit as needed.**
EOF

echo "✅ Handoff 생성: $HANDOFF_OUT"

# MEMORY update
MEMORY_FILE="/Users/crystal/.claude/projects/-Users-crystal-Desktop-brother/memory/reference_brother_pipeline.md"
if [ -f "$MEMORY_FILE" ]; then
  # 첫 줄 (다음 세션 시작 prompt) update
  cp "$MEMORY_FILE" "$MEMORY_FILE.bak"
  cat > "$MEMORY_FILE" <<EOF
---
name: brother-pipeline
description: RFP v$VERSION 다음 세션 시작 프롬프트 + mandatory 4 source
type: reference
---

# 다음 세션 시작 prompt
\`\`\`
cat $HANDOFF_OUT
\`\`\`

# Latest archive
$SESSION_DIR

# Mandatory 4 source (rfp-orchestrator v1.5)
- MOLIT 실거래가 (Worker A — brother-automation)
- KOSIS 인구·세대 (Worker A — API key 발급 권장)
- 청약Home 청약경쟁률 (Worker A — 웹 스크래핑)
- KB부동산 시세 (Worker A — 웹 스크래핑)
EOF
  echo "✅ MEMORY updated: $MEMORY_FILE"
fi

echo ""
echo "다음 세션 시작 명령:"
echo "  cat $HANDOFF_OUT"
```
